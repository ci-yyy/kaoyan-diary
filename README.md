## 项目简介

**考研日记**是一款面向中国研究生入学考试（考研）备考学生的本地化学习管理工具。它集成了**计时器、日记本、任务管理、目标追踪、数据统计**等核心功能，帮助考生：

- 📊 **量化每日学习** —— 分科目计时，直观看到时间分配
- 📝 **记录备考心路** —— 日记功能让你记录每一天的收获与感悟
- ✅ **管理每日任务** —— 待办事项清单，保持执行力
- 🎯 **设定目标分数** —— 设定院校、专业和科目目标分
- 📈 **追踪复习进度** —— 进度条、周统计、柱状图、科目分布图
- 🏆 **成就激励系统** —— 解锁成就徽章，保持学习动力

> 💡 **适用场景**：考研、考公、考证等任何需要长期备考的场景。

## 快速开始

### 方式一：双击启动（推荐）

项目自带 `start.command` 启动脚本：

1. 双击 `start.command`（macOS）或终端执行 `python3 server.py`
2. 浏览器自动打开 [http://localhost:8080](http://localhost:8080)
3. 开始使用！

### 方式二：直接打开（无服务器模式）

直接在浏览器中打开 `index.html` 文件即可使用。

> 所有数据自动保存在浏览器的 `localStorage` 中，无需服务器也能正常运行。

### 方式三：终端启动

```bash
cd /path/to/考研日记
python3 server.py
```

然后手动访问 [http://localhost:8080](http://localhost:8080)。

### 停止服务器

在终端按 `Ctrl + C` 即可停止服务。

## 功能详解

### 🎯 考研倒计时

页面右侧边栏顶部显示距考研初试（默认 **2026 年 12 月 19 日**）的倒计时：

- 剩余天数（大号醒目数字）
- 剩余周数
- 剩余小时数
- 考试日期提示

### 📖 四科计时器

预设考研四个核心科目，每科配有独立计时器：

| 科目 | Emoji | 满分 | 主题色 |
|------|-------|------|--------|
| 政治 | 📖 | 100 | 红色 `#c0392b` |
| 英语 | 📝 | 100 | 蓝色 `#2980b9` |
| 数学 | 🧮 | 150 | 绿色 `#27ae60` |
| 专业课 | 🔬 | 150 | 金色 `#c9a84c` |

**使用方式**：

- 点击任一科目卡片 → 开始计时
- 再次点击同一科目 → 暂停计时
- 点击其他科目 → 自动停止当前计时，切换到新科目
- 计时过程中可自由切换页面，不会丢失

> ℹ️ 计时器只在当天有效。查看历史日期时，计时功能自动隐藏。

### 📝 日记编辑

左侧为日记编辑区域，支持：

- 每日日记撰写（最多 10000 字）
- 实时字数统计
- 自动保存（失去焦点时）
- 手动保存按钮 / `Ctrl + Enter` 快捷键
- 显示当日励志名言（日期哈希索引，每天不同）
- 日记索引侧边栏（按月份分组，支持折叠/展开）

### ✅ 任务清单

右侧任务面板支持：

- 添加新任务
- 标记完成（点击圆圈）
- 删除任务
- 自动排序：未完成在前，已完成在后

### 🎯 目标设定

顶栏可设置：

- **目标院校** —— 你心仪的学校名称
- **目标专业** —— 报考专业
- **各科目标分** —— 政治、英语、数学、专业课分别设定
- **总分自动汇总**

### 📊 数据统计

#### 今日概览（侧边栏）

- 今日各科学习时长（实时更新）
- 今日总学习分钟数
- 总复习进度条（可设置总目标学时，默认 500 小时）
- 已学习天数
- 今日学习时间线（每条学习 session 的开始、结束、时长）

#### 本周统计

点击「📅 本周统计」展开：

- 按科目 × 七天的矩阵表格
- 每日各科学习时长
- 总计行
- **周柱状图** —— 每日总时长可视化
- **今日科目分布图** —— 各科学习时长占比（堆叠条形图 + 图例）

> 柱状图和分布图在开启周统计后自动渲染。

### 🏆 成就系统

内置 12 个成就，自动检测并弹出通知：

| 成就 | 条件 |

| 👑 一个月来风雨无阻 | 连续学习 30 天 |
| ⭐ 五十小时 | 累计学习 50 小时 |
| 🌟 破百小时 | 累计学习 100 小时 |
| 💎 三百小时 | 累计学习 300 小时 |
| ⚡ 五百小时大神 | 累计学习 500 小时 |
| 🚀 八小时日 | 单日学习 8 小时以上 |
| 🦾 十小时日 | 单日学习 10 小时以上 |
| 📚 日记十篇 | 写了 10 篇日记 |

### 🌙 深色模式

点击顶栏 🌙/☀️ 按钮切换深色/浅色主题，偏好自动保存。

### 📤 数据导出

支持两种导出格式：

- **JSON** —— 完整数据备份，包含所有结构，适合日后导入
- **TXT** —— 可读文本格式，直接打开查看，适合分享和打印

### 🔍 日记搜索

侧边栏搜索框支持按日记内容关键字实时过滤。

### 🧹 数据维护

- **清理旧数据**：自动清理 30 天前的学习 session 明细（汇总数据保留），释放存储空间
- **存储用量显示**：底部显示当前数据大小和日记天数

### 💬 励志语录

内置 70+ 条励志名言，每天切换一条，显示在日记编辑器右下角。

### 设计模式

| 方面 | 说明 |
|------|------|
| **架构模式** | 单页应用（SPA），前端驱动 |
| **UI 框架** | 无框架，纯原生 JavaScript |
| **渲染策略** | 脏标记（Dirty Flag）+ rAF 批处理 |
| **性能优化** | 轻量/全量双轨渲染、增量 DOM 更新 |
| **存储策略** | 双轨存储（localStorage + 服务端文件同步） |

### 性能优化（v8.5）

- **脏标记渲染**：只在标记变更时重绘，避免不必要的 DOM 操作
- **rAF 批处理**：`requestAnimationFrame` 合并多次更新为一次渲染
- **轻量/全量双轨**：计时器 tick 时只更新 textContent，不操作 innerHTML
- **增量会话更新**：时间线只更新活跃 session 的结束时间，避免全量重绘

### 数据存储（v8.6）

- **QuotaExceededError 处理**：存储满时自动清理旧 session 数据
- **30 天自动截断**：超期 session 明细自动删除，汇总数据保留
- **存储用量监控**：底部实时显示

### 无障碍访问（v8.7）

- 完整 ARIA 标注（`role`、`aria-label`、`aria-expanded`、`aria-current`）
- 键盘导航（方向键浏览日记列表，Enter/Space 激活）
- 焦点指示器（`:focus-visible`）
- 尊重系统 `prefers-reduced-motion` 设置

### 数据可视化（v8.8）

- **CSS 柱状图**：纯 CSS 实现，无第三方库依赖
- **堆叠条形图**：展示各科目学习时间占比
- **响应式布局**：适配移动端

## 设计理念

### 视觉风格

- **仿纸质感** —— 背景使用米白/暖灰色调 + 横线纹理，模拟笔记本纸面
- **Serif 字体** —— 使用 Georgia / Noto Serif SC / Songti SC 等衬线字体，营造阅读和书写氛围
- **克制配色** —— 以暖色系为主，四个科目各有主题色，便于视觉区分
- **温和过渡** —— 所有交互都有平滑过渡动画

### 交互设计

- **沉浸式书写** —— 编辑器背景与页面一致，消除边界感
- **即时反馈** —— 所有操作都有视觉反馈（通知、保存状态、进度条动画）
- **符合直觉** —— 点击科目即开始计时，点击任务即标记完成，零学习成本

### 存储策略

| 模式 | 存储位置 | 特点 |
|------|---------|------|
| 直接打开 `index.html` | `localStorage` | 离线可用，无需服务器 |

| Key | 用途 |
|-----|------|
| `ky_diary_state` | 应用状态（目标、任务、科目、设置） |
| `ky_diary_pages` | 日记内容 + 学习数据 |
| `ky_diary_theme` | 主题偏好 |

- `data.json` 位于项目根目录
- 将整个文件夹复制到其他电脑，通过 `server.py` 启动后自动读取
- **建议定期导出 JSON 备份**，防止误操作

## 无障碍支持

## 常见问题

### Q：数据存在哪里？会丢失吗？

数据同时存在浏览器的 `localStorage` 和（如果启动了服务器）`data.json` 文件中。建议定期使用「JSON 导出」功能备份数据。

### Q：如何迁移到另一台电脑？

将整个文件夹拷贝到新电脑，双击 `start.command` 或打开 `index.html` 即可。数据文件 `data.json` 会自动带入。

### Q：如何修改考研日期？

打开 `app.js`，找到第 9 行的 `EXAM_DATE` 常量，修改为你的目标考试日期：

```js
const EXAM_DATE = '2026-12-19'; // 改为你的目标日期
```

### Q：如何添加新科目？

目前支持四个固定科目。修改 `app.js` 中的 `DEFAULT_SUBJECTS` 数组（第 12-17 行）即可增删改科目。

### Q：开启服务器有什么优势？

启动 `server.py` 后：
1. 数据自动同步到 `data.json` 文件，即使清空浏览器缓存也不丢失
2. 可以通过文件系统直接查看和编辑数据

### Q：存储空间不足怎么办？

点击侧边栏的「🧹 清理旧数据」按钮，系统会自动删除 30 天前的学习记录明细（汇总数据不受影响）。如果仍不足，请导出 JSON 备份后清理 localStorage。
